Scope note (parity): all three variants model the same journeys — clinic defaults, per-visit-type flow/notes override, room membership — with the same data. Not ported in any variant: patient preview, operating-hours editor, form library management, and the live flow/notes builders themselves (each variant indicates where today's builder would embed unchanged). The Current pane is the real app.

Recommendation: B is the strongest long-term architecture — it is the only option that fixes cross-room duplication, and its per-section inherit/override rule is the easiest model to teach; pair it with A's reference semantics for flows & notes, and add C's matrix later as a reporting view on top. If migration cost dominates, A alone fixes drift and blast-radius today without moving anything. Your call.
